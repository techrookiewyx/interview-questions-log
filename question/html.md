## HTML部分

1. 如何在HTML中创建一个链接？
	- A：可以使用`<a>`标签来创建一个链接。如`<a href="https://www.example.com">Click here</a> `
	- 解析：`<a>`标签是HTML中用于创建超链接的标签，其最常见的属性为`href`用于用于指定链接的目标URL。
2. 如何在HTML中插入一张图片？
	- A：使用`<img>`标签来插入一张图片。例如：`<img src="image.jpg" alt="Image">`。
	- 解析：`<img>`标签是HTML中用于插入图片的标签。src属性用于指定图片的URL，alt属性用于指定图片的替代文本（当图片无法显示时会显示该文本）。
3. 如何在HTML中创建一个无序列表？
	- A：使用`<ul>`和`<li>`标签来创建无序列表。
	- 解析：在HTML中常用列表有`<ul>`无序列表、`<ol>`有序列表和`<dl>`自定义列表，`<li>`标签则表示列表中的列表项（ul和ol中使用）。
4. 请说一下HTML5有哪些新特性
	- A：HTML5是HTML的第五个版本，新特性包括语义化标签（如`header`、`<nav>`、`<footer>`等），表单增强（如新的输入类型、表单验证等）音视频支持（如`<video>`、`<audio>`标签）,本地存储（如localStorage、sessionStorage）等。
5. 请说一说HTML中的meta标签
	- A：`meta`标签用于提供一些关于HTML文档的元数据（描述数据的数据）。常见的`meta`标签包括charset（指定文档的字符编码）、viewport（指定视口的大小和缩放比例）、description（指定网页的描述信息）等。`meta`标签可以通过name和content属性来定义元数据。
		```html
		<!-- 定义文档编码字符集 -->
		<meta charset="utf-8">
		<!-- 设置视口的大小为300px -->
		<meta name="viewport"  content="Width:300px">
		```
	- 解析：meta标签是前端开发中常用的一种标签，meta标签的使用对于优化网页的SEO和用户体验非常重要。
6. 请解释一下HTML中的语义化标签。
	- A：语义化标签通俗来说就是可以直接知道该标签代表什么（指具有特定含义和作用的标签），可以通过语义化标签来划分网页的结构。由语义化标签构建的网页的页面的结构更加清晰、易于理解和方便维护，同时也有利于引擎对文档的解读、搜索引擎的优化、设备解析。
7. 网页抽象成三部分（HTML、CSS和JavaScript）后的优点有哪些？ 
	- A：三部分是HTML代表结构负责页面内容和结构、CSS代表表现负责内容样式、JS表示行为负责页面动态交互。
		抽象为三部分可以`增加代码可读性`、`按需加载资源`、`便于后期维护`。
8. HTML5删除和新增了哪些元素
	- A：新增了`article`、`header`等内容元素，`data`、`time`、`number`等表单元素，`video`、`audio`多媒体元素，`<canvas>`绘图元素等。移除了`<applet>`、`<basefont>`、`<big>`、`<center>`、`<font>`、`<frame>`等元素。
9. 页面导入样式时，使用 link 和 @import 有什么区别？
	- A：
		1. 从属关系区别。`link`属于HTML提供的标签，而`@import`是CSS提供的语法规则，只能导入样式文件的作用，而`link`不仅可以加载css文件，还可以定义`RSS`、`rel`链接属性、引入网站图标等。
		2. 加载顺序区别。页面加载时，`link`会同时被加载，而`@import`引用的CSS会等页面被加载完后再加载。
		3. 兼容性区别。`@import`只有IE5以上才能被识别，而link是HTML标签，无兼容问题。
		4. DOM可控性区别。`link`可以通过js操作DOM动态引入样式表改变样式，`@improt`而不可以。
10. 渲染过程中遇到 JS 文件怎么处理？
	- A：JavaScript 的加载、解析与执行会阻塞文档的解析，也就是说，在构建 DOM 时，HTML 解析器若遇到了 JavaScript，那么它会暂停文档的解析，将控制权移交给JS引擎，等JS引擎运行完毕，浏览器再从中断的地方恢复对文档的解析。
		如果你想让页面更快的加载，就应该将`<script>`标签放到body标签的底部这样就可以在页面加载完毕后再去加载JS文件，当然也可以使用script标签的defer或async属性来改变页面对JS的加载方式。
11. async 和 defer 的作用是什么？有什么区别？
	- A：脚本没有 defer 或 async，浏览器会立即加载并执行指定的脚本，也就是说不等待后续载入的文档元素，读到就加载并执行。
		1. defer属性：表示脚本可以延迟到文档完全被解析和显示之后再执行（脚本的加载和HTML解析是同时进行的）
		2. async属性：表示异步执行引入的脚本，与defer区别在于，如果脚本已经加载完毕则会立即执行（会阻塞文档解析）且无法保证执行顺序
12. b 与 strong 的区别和 i 与 em 的区别？
	- A： 从页面显示效果来看，被 `<b>` 和` <strong>` 包围的文字将会被加粗，而被` <i>` 和` <em> `包围的文字将以斜体的形式呈现。
		> `<b>` 和`<i>` 是自然样式标签，分别表示无意义的加粗，无意义的斜体，仅仅表示「这里应该用粗体显示」或者「这里应该用斜体显示」
		>
		> 而`<em>`和`<strong>`是语义样式标签。`<em>`表示一般的强调文本，而`<strong>`表示比`<em>`语义更强的强调文本
13. 说一说行内元素和块级元素，有什么区别？
	- A：
		> 行内元素只会占据它对应标签内容以及边框所占据的空间，常见的行内元素有 a  img strong sub span label等
		
		> 块级元素占据其父元素（容器）的整个宽度因此创建了一个“块”，常见的块级元素有 div ul li  h1~h6 p等
		
		> （1）格式上，默认情况下行内元素不会以新的一行开始，而每个块级元素会独占一行。
		>
		> （2）内容上，默认情况下行内元素只能包含文本和其他行内元素，而块级元素可以包含行内元素和其他块级元素。
		>
		> （3）行内元素在盒模型属性上无法设置width和height属性，设置 margin 和 padding 的上下不会对其他元素产生影响。
14. 简述超链接target属性的取值和作用
	- A：target属性一共有四种取值。
		- `__self`：默认值，在当前窗口或框架中加载目标页面。
		- `_blank`：在新的窗口或者标签页中打开目标页面，在使用这个属性时，最好添加 `rel="noopener norefferrer"` 属性，防止打开的新窗口对原窗口进行篡改。防止 `window.opener` API 的恶意行为。
		- `_parent`：在 `frame` 或者 `iframe` 中使用较多。在父级框架中载入目标页面，当 `a` 标签本身在顶层时，与 `_self` 相同。
		- `_top`：在 `frame` 或者 `iframe` 中使用较多。直接在顶层的框架中载入目标页面，加载整个窗口。
		- `任意字符`：若该链接不是已打开的页面，则在新窗口中打开，与_blank一致；若该链接已经打开，则跳转到该标签页并刷新页面
15. doctype的作用是什么？
	- A：首先doctype位置必须放在文档顶端（否则可能令早期的浏览器触发怪异模式）、DOCTYPE不是HTML标签、对大小写不敏感，其作用如下

		- 对文档进行解析：来告知浏览器的解析器用什么文档标准解析这个文档（防止浏览器渲染时切换到怪异模式），主要有以下三种方式解析HTML文档（标准模式、怪异模式、近乎模式）

		解析：`“标准模式”`下页面按照 HTML 与 CSS 的定义渲染，而`“怪异模式”`就是浏览器为了兼容很早之前针对旧版本浏览器设计、并未严格遵循 W3C 标准的网页而产生的一种页面渲染模式。
16. HTML5 为什么只需要写 `<!DOCTYPE HTML>`，而不需要引入 DTD？
	- A：HTML5 不基于SGML，因此不需要对DTD进行引用，但是需要 DOCTYPE 来规范浏览器的行为（让浏览器按照它们应该的方式来运 行）。
	- 解析：DTD（文档类型定义）是一组机器可读的规则，用来定义HTML或XML的特定版本中一系列合法的元素、属性和文档结构。简单来说就是用于定义文档，向浏览器说明使用什么样的渲染模式。
17. SGML 、 HTML 、XML 和 XHTML 的区别？
	- A：
		- SGML是标准通用置标语言，是一种定义文档结构和描述其内容的国际标准语言，是所有文档标记语言的起源。  
		- HTML是超文本标记语言，主要是用于规定怎么显示网页。 
		-  XML是可扩展标记语言是未来网页语言的发展方向，XML 和 HTML 的最大区别就在于 XML 的标签是可以自己创建的，数量无限多， 而 HTML 的标签都是固定的而且数量有限。  
		- XHTML指可扩展的超文本标记语言，它其实和 HTML 没什么本质的区别，标签都一样，用法也都一样。但它可以很好处理各大浏览器的兼容，比HTML更严格，比如标签必须都用小写，标签都必须有闭合标签且要在属性值上使用双引号等。
18. 介绍一下你对浏览器内核的理解？
	- A：浏览器内核主要分为两个部分，分别是渲染引擎和JS引擎。
		> 渲染引擎的职责就是渲染，即在浏览器窗口中显示所请求的内容。默认情况下，渲染引擎可以显示 html、xml 文档及图片，它也
		> 可以借助插件显示其他类型数据，例如使用 PDF 阅读器插件，可以显示 PDF 格式。
		>
		> JS 引擎：解析和执行 javascript 来实现网页的动态效果。
		>
		> 
		> 最开始渲染引擎和 JS 引擎并没有区分的很明确，后来 JS 引擎越来越独立，内核就倾向于只指渲染引擎。
19. 前端需要注意哪些 SEO ？
	- A：
		- 合理的`title`、`description`、`keywords`，搜索对着这三项的权重逐个减小。不同页面的title和description不同，keywords要靠前且重要关键字不要出现超过2次
		- 语义化的 HTML 代码，符合 W3C 规范：语义化代码让搜索引擎容易理解网页。
		- 重要内容 HTML 代码放在最前，保证搜索引擎可以抓取到
		- 重要内容不要用 js 输出
		- 非装饰性图片必须加 alt
		- 少用iframe以及提高网站速度
20. 说说你对html中的置换元素和非置换元素的理解
	- A：
		置换元素：内容不受CSS视觉格式化控制，CSS渲染引擎不考虑对此内容的渲染，且元素本身一般拥有固有尺寸的元素，称为置换元素
		
		> 简单来说置换元素就是浏览器根据元素的标签和属性，来决定元素的具体显示内容。浏览器根据img标签的src属性显示图片当src不同时显示图片的内容和尺寸是不同的，以及input标签根据type属性决定显示输入框还是按钮
		>
		> 常见的置换元素有img, input, textarea, select，video等
		
		非置换元素：除了置换元素外其他元素都是非置换元素，内容直接展示给浏览器
21. Label 的作用是什么？是怎么用的？
	- A： label 标签用来定义与表单控件间的关系，当用户选择点击label标签内文本时，浏览器会自动将焦点转到和标签关联的表单控件上。

		```html
		<!-- 将label标签的for属性的值设置为对应表单元素的id，来产生关联 -->
		<label for="Name">Number:</label>
		<input type=“text“ name="Name" id="Name"/>
		```
22. 什么是重绘和回流？
	- A：
		- 重绘：当文档中的元素需要更新属性，而这些属性只是影响元素的外观、风格，而不会影响布局的操作，比如 background-color，我们将这样的操作称为重绘。

			> 常见的引起重绘的属性：color、visibility、background、text-decoration、outerline、boder-radius、box-shadow等

		- 回流：当文档中一部分（或全部）因为元素的规模尺寸、布局、隐藏等改变，而需要重新构建的操作（或者是影响页面布局的操作），这样的操作称为回流。
		
			> 常见的引起回流的情况：
			>
			> - 页面首次渲染
			>
			> - 添加或者删除可见的 DOM 元素；
			> - 元素尺寸改变或位置改变——边距、填充、边框、宽度和高度
			> - 元素内容变化（文字数量或图片尺寸等）
			> - 浏览器窗口尺寸改变——resize事件发生时
			> - 查询某些属性或调用某些方法
			> - 激活CSS伪类
			> - 元素字体大小变化
			>
			> 常见方法和属性：width、height、margin、padding、border、position、display、overflow、clientTop/Btoom()、offsetWidth/Height()、scrollWidth/Height、getComputedStyle()等
		
		回流必定会发生重绘，重绘不一定会引发回流。回流所需的成本比重绘高的多，改变父节点里的子节点很可能会导致父节点的一系列回流。
23. 如何减少回流？
	- A：可以从CSS和JS两个方面分析

		> 在CSS部分遵循以下几点：
		>
		> - 避免使用 table 布局，可能很小的一个小改动会造成整个 table 的重新布局
		> - 尽可能在DOM树的最末端改变class
		> - 避免设置多层内联样式
		> - 将动画效果应用到position属性为absolute或fixed的元素上
		> - 避免使用CSS表达式（例如：calc()）

		> JS方面：
		>
		> - 不要一条一条地修改 DOM 的样式。与其这样，还不如预先定义好 css 的 class，然后修改 DOM 的 className。
		> - 避免频繁操作DOM，使用 documentFragment 对象在内存里操作 DOM
		> - 避免频繁读取会引发回流/重绘的属性，不要把节点的属性值放在一个循环里当成循环里的变量
24. title 与 h1 的区别？
	- A：title 属性没有明确意义只表示是个标题，h1 则表示层次明确的标题，对页面信息的抓取也有很大的影响。
25. 网页验证码是干嘛的，是为了解决什么安全问题？

		- 有效防止黑客对某一个特定注册用户用特定程序暴力破解方式进行不断的登陆尝试
26. 用于预格式化文本的标签是？
	- A：\<pre>标签定义预格式文本，保持文本原有的格式
	- 解析：预格式化就是保留文字在源码中的格式，最后显示出来样式与源码中的样式一致所见即所得。
27. 渐进增强和优雅降级的定义
	- A：
		- 渐进增强：针对低版本浏览器进行构建页面，保证最基本的功能，然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验。   
		- 优雅降级：一开始就根据高版本浏览器构建完整的功能，然后再针对低版本浏览器进行兼容。
28. \<img> 的 title 和 alt 有什么区别？
	- A： title 通常当鼠标滑动到元素上的时候显示。 alt 是 \<img> 的特有属性，是图片内容的等价描述，用于图片无法加载时显示、读屏器阅读图片。可提高图片可访问性，除了纯装饰图片外都必须设置有意义的值，搜索引擎会重点分析。


30. attribute 和 property 的区别是什么？
	- A：

		- attribute是dom元素在文档中作为html标签拥有的属性
		- property 就是dom元素在js中作为对象拥有的属性

		对于 html 的标准属性来说，attribute 和 property 是同步的，是会自动更新的，  但是对于自定义的属性来说，他们是不同步的

32. disabled 和 readonly 的区别？
	- A：

		> disabled 指当 input 元素加载时禁用此元素。input内容不会随着表单提交。     
		>
		> readonly 规定输入字段为只读。input内容会随着表单提交。   
		>
		> 无论设置 readonly 还是 disabled，通过 js 脚本都能更改 input 的 value
33. 常见的浏览器端的存储技术有哪些？
	- A：浏览器常见的存储技术有 cookie、localStorage 和 sessionStorage，还有两种存储技术用于大规模数据存储，webSQL（已被废除）和 indexDB，IE支持userData 存储数据，但是基本很少使用到，除非有很强的浏览器兼容需求
34. HTML5 的 form 的自动完成功能是什么？
	- A： autocomplete属性规定输入字段是否应该启用自动完成功能。默认为启用，设置为 autocomplete=off 可以关闭该功能。   

		> 自动完成允许浏览器预测对字段的输入。当用户在字段开始键入时，浏览器基于之前键入过的值，应该显示出在字段中填写的选项，类似于历史记录一样。
		>
		> autocomplete属性适用于\<form>，以及下面的\<input> 类型为：text, search, url, telephone, email等。
35. iframe 有那些缺点？
	- A： iframe 元素会创建包含另外一个文档的内联框架（即行内框架）。

		> 主要缺点有：   
		>
		> 1. iframe 会阻塞主页面的 onload 事件。window 的 onload 事件需要在所有 iframe 加载完毕后（包含里面的元素）才会触发。
		> 2. 搜索引擎的检索程序无法解读这种页面，不利于网页的 SEO 。  
		> 3. iframe 和主页面共享连接池，而浏览器对相同域的连接有限制，所以会影响页面的并行加载。  
		> 4. 浏览器的后退按钮失效。  
		> 5. 小型的移动设备无法完全显示框架。
36. 什么是文档的预解析？
	- A： Webkit 和 Firefox 都做了这个优化，当执行 JavaScript 脚本时，另一个线程解析剩下的文档，并加载后面需要通过网络加载的资源。这种方式可以使资源并行加载从而使整体速度更快。需要注意的是，预解析并不改变 DOM 树，它将这个工作留给主解析过程，自己只解析外部资源的引用，比如外部脚本、样式表及图片。
37. CSS 如何阻塞文档解析？
	- A：理论上，样式表不改变DOM树，也就没有必要停下文档的解析等待他们，但JS脚本执行时可以在文档的解析过程中请求样式信息，如果样式还没有加载和解析，脚本将得到错误的值。所以如果浏览器尚未完成CSSOM（浏览器解析CSS，生成的规则树）的下载和构建，而我们此时想要运行脚本，那么浏览器将延迟JS脚本执行和文档解析，直至其完成CSSOM的下载和构建。也就是说在这种情况下，浏览器会先下载和构建 CSSOM，然后再执行 JavaScript，  最后再继续文档的解析。
38. 空元素定义
	- A：标签内没有内容的 HTML 标签被称为空元素。空元素是在开始标签中关闭的。常见的空元素有：br、hr、img、input、link、meta
